超频教程 - HOC

免责声明

请自行承担任何由超频带来的风险与后果!!!
超频可能导致包括但不限于异常耗电, 发热, 不稳定, 资料丢失, 系统及硬件损坏等等问题.

注意事项

请先行做好备份, 避免重要资料丢失.
尽量避免 使用 exFAT格式的内存卡 进行折腾, 可以参考 教程
尽量避免 SysMMC真实系统 折腾!!!

支持 4G/8G 内存, 8G 性能相对较弱.
跨版本使用不对应版本的 kip 会导致无法进入系统.
跨版本导入旧超频配置可能导致 kip 文件污染, 系统崩溃等问题.

有条件情况下, 使用副卡折腾测试, 再将稳定参数应用到主卡.

查看硬件

重启按住 <音量键减号> 进入 hekate 主菜单
打开顶部第三个 Console Info > HW & Fuses
记录以下信息:

  • DRAM ID: 内存型号
  • CPU/GPU/SOC Speedo: 体质值

预设配置

  • 预设 G2400, G2666 两个通用配置
  • 可通过 Preferences 参数 > Manage Presets 管理参数 > ... 导入配置, 重启后生效

调试顺序

  1. MEM 内存参数
  2. TIMINGS 时序参数
  3. GPU 参数
  4. CPU 参数

未完全确定参数稳定之前, 不要开始下一步的操作. 新手每一次只动一个参数, 测试没问题之后开始再下一个参数.

MEM 内存参数

DVB 核心电压: 0-10 档

内存频率越高, Soc 电压要求越高
25 mV 每档, 新手根据自己目标频率, 先参考 电压计算器 的估算结果
同参数因 Speedo 差异, 会出现 Soc 电压差异, 请以 Sys-clk 实时 Soc 实际电压为主

VDD2/VDDQ 内存电压: 提高适当电压可以提高稳定性, 可能达到更高的内存频率, 但不是突破频率上限的因素.

1866BL 可以提升性能, 但是可能会降低内存频率上限, 可能需要额外加压.
1333BL 会明显降低内存频率上限(300-500MHz), 且 t2 <=2, 整体电压要求较高, 理论上性能最好.
2533 内存频率相比其他频率, 需要更宽松的时序.

TIMINGS 时序参数

核心时序: t1 tRCD, t2 tRP, t3 tRAS, t8 tREFpb

  • 核心时序决定内存表现基础
  • 降低核心时序会降低性能, 降低可能跑到更高内存频率

极限时序: 通常能获得比通用时序更好的性能, 但可能需要更好的体质, 或者更高的电压

建议在开始调整测试时序之前, 先找出可用的最大内存频率, 可以降低后续排查的难度

  • 初次摸索极限时序, 建议 DVB 10, 排除 Soc 电压不足的问题
  • 先测试预设的极限时序
    • 如果无法正常进入系统, 重启按住 音量键 减号 进入 hekate 主菜单
    • hekate > Launch > Restore 启动到恢复模式, 调整参数.
    • 推荐降低时序顺序: t1 → t2 → t3 → t6 → t7 → t4 → t5
    • 切记不要一次性调整多个参数, 导致无法排查问题点
  • 参数初步稳定后, 建议参考 测试方法 , 进行相关的测试以确认参数稳定性.

HP Mode 高性能模式

  • 默认: 关闭
  • 开启一定程度上可以提高游戏表现, 极限状态可能需要额外的电压

Presets 预设时序

  • 包含 CT 通用时序, ST 极限时序
  • 不推荐直接使用 ST 极限时序, 仅作为参考范围
  • 使用前一定要确认好自己的内存颗粒, 不支持跨型号使用

GPU 参数

UV Level 降压等级:

  • 范围: 0-2
  • 等级越高, 降压越多, 可安全使用频率越多
  • Speedo < 1650 UV 1
  • Speedo > 1650 UV 2
  • 自定义电压设定值会覆盖 UV 的默认电压值, 可以允许更多的操作空间, 尤其 Speedo > 1700 的机器

Volt Offset 电压偏移值

  • 影响 Auto 自动 的自定义整体电压

Vmin 最小电压

  • 最低 450mV
  • 影响 DVFS 模式 的电压
  • Hijack 操控模式
    • 开启: 影响 最小 内存频率的GPU电压
    • 禁用: 影响 最大 内存频率的GPU电压

Vmax 最大电压

  • 影响可用频率范围
  • 超过最大电压的频率会被隐藏
  • 安全范围 <= 800mV

GPU Boost 提高GPU占用率上限

  • 最多可以 99.6% -> 99.9%

Custom Table 自定义电压

  • Disabled 禁用 被禁用频率以上的都会隐藏

CPU 参数

Boost CPU 睿频频率

  • 不等同于 Sysclk 中设定的目标频率
  • 仅控制系统调用加载模式的工作频率
  • 频率越高, 性能越强, 功耗也越高
  • 默认: 1963 MHz (建议: 2397 Mhz)

UV Level 降压等级

  • 控制所有频率的电压偏移值
  • 选项越高, 降压幅度越大(建议: 先后尝试 4, 8 两档)

High UV Level 高频降压

  • 控制 Table Mode 控压模式 起始频率以上的电压偏移值
  • 选项越高, 降压幅度越大, 覆盖 UV Level 降压等级 的高频电压偏移值
  • 至少需要 1 档才能解锁的最高的睿频上限(建议先后尝试 4, 8 两档)

Low Vmin 低频最小电压

  • 最小电压: 550mV
  • 受体质影响, 过低的电压会导致无法正常进入系统

High Vmin 高频最小电压

  • 影响 Table Mode 控压模式 的起始电压计算
  • 过低的电压会导致死机, 软件报错

Vmax 最大电压

  • 默认: 1120 mV
  • 影响所有频率的电压计算
  • 过低过高的电压会导致硬件损坏, 不建议任何人使用默认值以外的设置

Table Mode 控压模式

  • 默认: 1683 MHz
  • 控制额外加压的起始频率
  • 如果遇到 UV 不能正常使用的情况, 可以尝试 1581 MHz 档位
  • 受 High Vmin 高频最小电压 影响

CPU Unlock 解锁频率

  • 解锁额外的可用频率 (仅限初代)

Max Clock 最大可用频率

  • 解锁额外的可用频率
  • 可选范围: 1963~3009MHz
  • 默认: 1963MHz

测试方法

使用底座模式进行游戏压力测试, 加快测试效率, 不要使用内存跑分测试工具 (如: Memtoolkit)
